## 프로젝트 개요

제시어를 보고 **똑같은 동작**을 취하세요. 평상시에는 서로의 실제 카메라 화면을 볼 수 없고, 오직 **텔레파시 게이지**와 AI MC의 코칭만으로 동작을 맞춥니다. 단, 한 제시어에서 오래 막히면 카메라 힌트가 잠깐 공개되어 다시 동작을 맞출 수 있습니다.

본 프로젝트는 라즈베리파이 기반 카메라 워커, FastAPI 서버, 브라우저 참가자/관객 화면으로 구성된 **온디바이스 AI 파티 게임**입니다. 각 카메라 워커는 로컬에서 MediaPipe Pose Lite로 포즈 랜드마크를 추정하고, 서버는 여러 참가자의 얼굴 방향과 상체 bone 벡터 유사도 및 활동성을 계산해 실시간 텔레파시 점수를 제공합니다. 게임은 60초 스프린트 방식으로 진행되며, 참가자들이 같은 동작을 취해 게이지 90점 이상을 달성하면 해당 제시어를 클리어하고 다음 제시어로 넘어갑니다. 로컬 LLM/정적 폴백 기반 진행자는 한국어 진행 멘트를 제공하며, 네트워크가 제한된 환경에서도 게임 핵심 기능이 동작하도록 폴백 경로를 제공합니다.

## 명세 작성 기준

- 요구사항 ID는 기능/비기능을 미리 분류하지 않고 `R-01`, `R-02` 형식의 중립 번호를 사용합니다.
- 검증 기준은 평가자 또는 평가 Agent가 완료 여부를 판단할 수 있도록 화면, 로그, API 응답, 설정값, 파일 산출물 기준으로 작성합니다.
- 시연 영상으로 확인 가능한 항목은 `MM:SS` 형식의 타임스탬프를 적고, 영상으로 확인하기 어려운 항목은 `N/A (사유)`로 표기합니다.

## 증빙 및 검증 메모

- 영상 시연 항목은 최종 시연영상의 해당 타임스탬프에서 화면 동작을 확인합니다.
- 2026-06-30 기준 명세는 5라운드 평균 방식이 아니라 60초 스프린트/90점 클리어/20초 제시어 제한/10초 카메라 힌트 방식의 최신 구현을 기준으로 작성했습니다. 최종 시연영상 재촬영 또는 편집 후 타임스탬프를 실제 영상 기준으로 다시 확인해야 합니다.
- `N/A` 항목은 하드웨어 구성, 로그, 설정 파일, 장애 주입, 네트워크 조건처럼 영상만으로 판정하기 어려운 항목이며 실행 로그·README·설정 파일로 보조 증빙합니다.
- 결과파일에는 서버/카메라 워커 소스, `requirements.txt`, 실행 스크립트, 설정 예시, Git 히스토리 또는 `git log` 결과를 포함해야 합니다.
- `edge-tts`는 외부 Microsoft TTS 서비스를 사용할 수 있으므로 완전 오프라인 검증에서는 브라우저 Web Speech 또는 정적 텍스트 폴백을 성공 기준으로 봅니다.
- R-26 오프라인 LAN 검증은 하나의 공유기를 내부 LAN으로만 사용하고 WAN/인터넷을 차단한 뒤, 서버 LAN IP 기준으로 카메라 워커와 브라우저가 연결되어 60초 스프린트 1회를 종료까지 진행하는 방식으로 보여줍니다. 상세 절차는 `test-results/offline-lan/R-26-offline-lan-guide.md`에 작성했습니다.
